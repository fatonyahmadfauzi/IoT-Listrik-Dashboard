sensorsSistem & Sensor
Logika baru: NORMAL < threshold × (peringatan%). WARNING antara peringatan% dan 100%. DANGER jika arus ≥ threshold.
Range rekomendasi: 1 – 100 A.
Contoh 80 → WARNING dari 80% threshold sampai < 100%.
Seberapa sering ESP32 menulis data ke Firebase.
Min: 500 ms | Rekomendasi: 2000 ms.

Buzzer

Bunyi saat kondisi abnormal terdeteksi

Auto-Cutoff Relay

Relay dimatikan otomatis saat DANGER (arus ≥ threshold)

tuneKalibrasi Sensor
Prosedur kalibrasi: bandingkan pembacaan ESP32 dengan alat ukur referensi.
Faktor = nilai_referensi ÷ nilai_raw_ESP32
Nilai default: 1.000 (tidak ada koreksi).
Contoh: clamp meter baca 5 A, ESP32 baca 4.8 A → faktor = 5/4.8 = 1.042
Faktor konversi ADC → Volt AC.
Contoh: voltmeter = 220 V, raw ADC RMS = 0.336 V → faktor ≈ 654.8
Untuk hitung daya (W) dan kWh di firmware — bukan pengukuran PF sejati.

wifi Bootstrap Device & Wi-Fi ESP32
Khusus admin utama: bagian ini mengirim konfigurasi bootstrap ke device fisik melalui Firebase. ESP32 akan menyimpan SSID, password Wi-Fi, API Key, URL Realtime Database, dan akun IoT ke NVS lalu restart otomatis.
Akun temp simulator tidak memakai bagian ini.
Device akan mencoba SSID ini lebih dulu saat boot. Jika gagal, ESP32 akan kembali membuka captive portal.
Saat menekan Simpan & Terapkan, device fisik akan membaca permintaan ini, menyimpan konfigurasi baru ke NVS, lalu restart otomatis. Bila menekan Hapus Wi-Fi & Buka Setup, device akan menghapus bootstrap saat ini dan masuk ke mode captive portal.
ink_eraser Reset Data Realtime IoT
Khusus admin utama: aksi ini akan mengosongkan data realtime sensor pada node /listrik tanpa menghapus histori log.
Jika device fisik masih online, data baru dapat muncul lagi pada heartbeat berikutnya.
Alur verifikasi: ketik nama project secara persis untuk menyetujui pengosongan data realtime perangkat IoT.
Konfirmasi harus sama persis dengan nama project agar aksi reset realtime dapat dijalankan.
Nilai arus, tegangan, daya, energi, frekuensi, power factor, relay, dan status realtime akan direset ke nilai kosong/default pada /listrik.
backup Backup Database Firebase
Khusus admin utama: aksi ini membuat salinan database Realtime Database dalam format JSON dan melampirkan file database.rules.json aktif dari project workspace ini ke email admin.
Backup ini tidak mengubah data apa pun di Firebase.
Cocok dipakai sebelum reset atau hapus data monitoring. File yang dikirim berisi snapshot RTDB saat tombol dijalankan, sehingga kamu punya arsip sebelum melakukan perubahan besar.
Email akan berisi dua lampiran: default-rtdb-export.json dan database-rules.json.
Hanya membuat dan mengirim backup. Data realtime, histori log, settings, dan user tidak disentuh.
delete_sweep Hapus Semua Data Monitoring
Khusus admin utama: aksi ini akan mengosongkan seluruh data monitoring perangkat IoT pada /listrik dan menghapus histori log pada /logs.
Pengaturan sistem, data user, dan bootstrap device tidak ikut dihapus.
Alur verifikasi: klik kirim OTP, buka email admin yang sedang login, lalu masukkan 6 digit kode untuk menyetujui penghapusan semua data monitoring perangkat IoT.
OTP hanya berlaku beberapa menit dan hanya dapat dipakai untuk satu kali hapus semua data monitoring.
Data realtime pada /listrik akan dikosongkan ke nilai default, lalu histori log pada /logs akan ikut dihapus sampai kosong.
cloud_sync Backend & Failover (klien web)
Disimpan di browser ini saja (localStorage) — ESP32 tidak membaca bagian ini. Mode LOCAL memerlukan backend-local berjalan. Browser tidak bisa auto-start server — gunakan aplikasi Windows.
